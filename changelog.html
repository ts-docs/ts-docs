Bug fixes

  • Computed property names can no longer be any - if ts-docs cannot find the provided type, then it uses the stringified version of the name.
  • Fixed a bug that's been tornmenting me for months: Some modules didn't have any exports for "mysterious" reasons.

Changes

  • Type parameters are now colored differently.
  • Bottom border of h elements is now darker and less noticable.
  • Search results now show instantly.
  • Search results are limited to 100.
  • Search results are now guaranteed to be in 2 columns.
  • Search results are now ordered in a way which makes more sense. Previously, the first half of the results was on the left colunm, the other half on the right. Now, the results alternate. The first result is on the left, the second on the right, etc.
  • If there isn't a 100% match, classes / interfaces / enums / functions / types / constants will appear above properties and methods.
  • More default APIs are now being referenced.

Features

  • Ts-docs now allows you to run unit tests inside documentation comments. If the docTests option is turned on, all method / function codeblocks inside documentation comments will be transpiled and ran. This feature is very similar to rustdoc's documentation tests. You can read more about this option in the guide.
  • A test option to go along with docTests. Using test allows you to only run certain tests. It also automatically turns on docTests and forceEmit.
  • A new custom pages front matter setting: redirect - It makes the custom page redirect you to the provided link.
