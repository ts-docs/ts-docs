Changelog

Tag v0.4.0, by GoogleFeud, released on 12/03/2022, 14:26:39

Download .zip

Download .tar

Bug fixes

  • Extractor would error if you were to export a constant with the deconstructing pattern:
// Now works!
export const { A, B } = ...

Changes

  • Exported constants whose value is a function are now treated as functions and not constants.
  • Errors which come from ts-docs are prettier.
  • Param text is no longer in bold.
  • If there are more than 6 dropdowns inside the sidebar, all of the dropdowns will be closed by default.
  • Mapped type modifiers are now shown in the documentation.
  • Template literal types are now rendered a bit differently - the syntax for expressions (${}) is now included.
  • Made sidebar category names significantly smaller.
  • Constant declarations are now split into two parts: one for the type of the constant, and one for it's content (stringified initializor)

Additions

  • Added Capitalize / Uncapitalize / Uppercase / Lowercase external types.

  • Object literal properties, tuple values and function/method parameters can now have comments:

    export function test(
    /** This is a function parameter comment */
    values: [
      name: string,
      value: number,
      /** Tuple value comment - only named values can have comments */
      deps: Array<string>
]) {
    return {
        /** Object property comment */
        result: "OK",
        code: 201
    }
}

    Renders the following:

  • You can now use | instead of as to rename references. (compatibility with typedoc and tsdoc)

  • Full support for the @link tag.

  • Support for the @remarks tag.

  • Support for the @inheritDoc tag.

  • Support for the @throws tag.

  • Support for the @alpha tag.

  • Support for the @experimental tag.

  • Readme files inside sub-directories are now added to the module's description.

  • You can now search for a reference in markdown by it's type ([[MyType:class]] for example)

  • You can now reference properties and methods of a class/interface/enum without prefixing them with the class/interface/enum's name.

  • Warnings are emitted if a reference in markdown cannot be found.

  • Ability to group items in categories via the @category tag:

    /**
 * @category Custom Category
 */
export const test = <T>(values: [

    Results in this getting rendered inside the sidebar only at the root of the project:

    This allows you to make navigation easier by displaying important deeply-nested items at the landing page of the documentation.

  • A logNotDocumented option, which logs all declarations (classes, interfaces, enums, functions, types, constants) which don't have any documentation comments.

  • version CLI flag which shows you the ts-docs version, typescript version and node.js version.

  • You can now use the @ignore tag to hide a function/method parameter.
